# 技术文档

## 项目概述

项目目标是把 Markdown 学习笔记转换为 Anki Front/Back 问答卡片包。脚本会读取本地 `.md` 文件，按标题和长度分块，调用兼容 OpenAI Chat Completions 的接口生成卡片 JSON，再导出为 `.apkg` 文件。

## 当前实现

- 语言：Python
- 核心依赖：
  - `genanki`：生成牌组、模型和 `.apkg`
  - `requests`：调用兼容 Chat Completions 的 HTTP API
  - `python-dotenv`：加载 `.env`
- 默认模型：`kimi-k2.5`
- 默认接口：`https://api.moonshot.cn/v1/chat/completions`

## 卡片结构

生成的每张卡片包含以下字段：

- `front`：正面问题
- `back`：背面答案
- `extra`：可选补充说明
- `source`：来源文件名
- `tags`：标签数组

Anki 模型字段为：

- `Front`
- `Back`
- `Extra`
- `Source`

## 核心流程

1. 加载 `.env` 并校验 `LLM_API_KEY` / `LLM_BASE_URL`
2. 读取 Markdown 文件
3. 按标题切块；超长文本按句子和长度继续拆分
4. 调用 LLM 生成 JSON 数组
5. 清理代码块包裹和额外文本
6. 校验每张卡片必须包含 `front` / `back`
7. 写入 Anki 牌组并导出 `.apkg`

## 错误处理

- 缺失配置时，启动阶段直接报错
- HTTP 请求失败时，打印明确的请求错误
- 模型返回非法 JSON 时，打印截断后的响应内容
- 卡片缺少 `front` / `back` 时，拒绝该批结果并提示具体原因

## 配置项

`.env` 支持以下字段：

```env
LLM_API_KEY=your_api_key_here
LLM_BASE_URL=https://api.moonshot.cn
LLM_MODEL=kimi-k2.5
LLM_TIMEOUT=120
```
